# 全站中英文国际化设计稿

Date: 2026-06-26
Scope: Hyperscaler Asset Hub (web)
Status: draft

---

## 1. 概述

当前系统所有页面的文本均为硬编码，公共页面以英文为主，后台管理页面中英混用。本次改造目标：

1. 全站支持中英文 URL 路由切换（`/zh/...`、`/en/...`）
2. 英文为默认语言，访问 `/` 重定向到 `/en`
3. 使用 `next-intl` 库实现 Next.js App Router 原生国际化
4. 所有硬编码字符串提取到翻译文件
5. 提供语言切换器组件（Header + Admin 面板均可用）

---

## 2. 技术方案

### 2.1 库选择：`next-intl`

- Next.js 16 App Router 官方推荐的 i18n 方案
- 支持 middleware 路由、Server Components、Client Components
- 支持命名空间（namespace）组织翻译文件
- TypeScript 类型安全

### 2.2 路由结构

当前：
```
web/src/app/
  layout.tsx          # RootLayout
  page.tsx            # Homepage
  about/page.tsx
  assets/page.tsx
  admin/page.tsx
  ...
```

改造后：
```
web/src/app/
  [locale]/
    layout.tsx          # RootLayout（含 NextIntlClientProvider）
    page.tsx            # Homepage
    about/page.tsx
    assets/page.tsx
    admin/page.tsx
    ...
```

所有页面和布局文件移入 `[locale]` 目录，middleware 负责：
- 检测用户语言偏好
- 无 locale 前缀的请求重定向到 `/en/...`
- 设置 `locale` cookie 记住用户选择

## 4. 核心实现细节

### 4.1 `next.config.ts`

```typescript
import createNextIntlPlugin from "next-intl/plugin";

const withNextIntl = createNextIntlPlugin("./src/i18n/request.ts");

const nextConfig: NextConfig = {
  images: {
    remotePatterns: [
      { protocol: "https", hostname: "**" },
      { protocol: "http", hostname: "**" },
    ],
  },
};

export default withNextIntl(nextConfig);
```

### 4.2 `src/i18n/routing.ts`

```typescript
import { defineRouting } from "next-intl/routing";

export const routing = defineRouting({
  locales: ["en", "zh"],
  defaultLocale: "en",
});
```

### 4.3 `src/i18n/request.ts`

```typescript
import { getRequestConfig } from "next-intl/server";
import { hasLocale } from "next-intl";
import { routing } from "./routing";

export default getRequestConfig(async ({ requestLocale }) => {
  const requested = await requestLocale;
  const locale = hasLocale(routing.locales, requested)
    ? requested
    : routing.defaultLocale;

  return {
    locale,
    messages: (await import(`../../messages/${locale}.json`)).default,
  };
});
```

### 4.4 `src/middleware.ts`

```typescript
import createMiddleware from "next-intl/middleware";
import { routing } from "./i18n/routing";

export default createMiddleware(routing);

export const config = {
  matcher: ["/((?!api|_next|.*\\..*).*)"],
};
```

### 4.5 `src/app/[locale]/layout.tsx`

```typescript
import { NextIntlClientProvider, useMessages } from "next-intl";
import { notFound } from "next/navigation";
import { hasLocale } from "next-intl";
import { routing } from "@/i18n/routing";

export default async function RootLayout({
  children,
  params,
}: {
  children: React.ReactNode;
  params: Promise<{ locale: string }>;
}) {
  const { locale } = await params;
  if (!hasLocale(routing.locales, locale)) notFound();

  return (
    <html lang={locale} suppressHydrationWarning className={...}>
      <body className={...}>
        <NextIntlClientProvider>
          <ThemeProvider>
            <QueryProvider>{children}</QueryProvider>
          </ThemeProvider>
        </NextIntlClientProvider>
      </body>
    </html>
  );
}
```

### 4.6 语言切换器 `src/components/locale-switcher.tsx`

```typescript
"use client";

import { usePathname, useRouter } from "@/i18n/navigation";
import { Globe } from "lucide-react";

export function LocaleSwitcher({ locale }: { locale: string }) {
  const pathname = usePathname();
  const router = useRouter();

  function switchLocale(target: string) {
    router.replace(pathname, { locale: target });
  }

  return (
    <button
      onClick={() => switchLocale(locale === "en" ? "zh" : "en")}
      className="inline-flex items-center gap-1.5 text-sm text-muted-foreground hover:text-foreground"
    >
      <Globe className="size-4" />
      {locale === "en" ? "中文" : "EN"}
    </button>
  );
}
```
